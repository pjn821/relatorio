
## fds
    Metodogias de Ds Tradicionais/Classicas

        Cascata: sequencial, rígido, para requisitos bem definidos e estáveis. Bom para sistemas críticos. Menos flexível às mudanças, risco de detectar problemas tardiamente.

        Prototipagem: iterativa, para requisitos incertos ou em mudança. Foca em feedback do usuário e testes prévios. Flexível, ideal para UI e projetos dinâmicos.

        Quando usar:
        Cascata quando a estabilidade é garantida; 
        Prototipagem para maior adaptação e interação com o cliente.
